## Erledigte Anpassungen

### 1. SSN/EIN Normalisierung (vision.py)

**Problem:** OCR liest SSN/EIN manchmal im falschen Format (z.B. `12-3456789` statt `123-45-6789`). Kein CSV-Treffer moeglich.

**Fix:** Bestehenden Leading-Zero-Fix an **zwei Stellen** in `vision.py` ersetzt:

- `read_all_rows_from_screenshot()` (~Zeile 897)
- `scan_visible_clients_csv()` (~Zeile 989)

Neuer Code (ersetzt den alten `re.match` Block):
```python
# Normalize SSN/EIN to XXX-XX-XXXX format
digits = re.sub(r"[^0-9]", "", ssn_ein)
if len(digits) == 8:
    digits = "0" + digits
if len(digits) == 9:
    ssn_ein = f"{digits[:3]}-{digits[3:5]}-{digits[5:]}"
```

**Logik:**
1. Alle Nicht-Ziffern entfernen
2. 8 Ziffern → Leading Zero ergaenzen
3. 9 Ziffern → im Format XXX-XX-XXXX formatieren

**Status:** Nur lokal auf TM55 implementiert. Noch nicht in Git.

### 2. CSV Spaltenkoepfe

**Problem:** Manuell erstellte CSV hatte Spalte `SSN/EIN` statt `ID`.

**Fix:** Spalte in CSV umbenannt zu `ID` (erwartet: `Client,ID,Return Type,Status`).

### 3. settings.json kalibriert

TM55 hat eine eigene `settings.json` mit angepassten Koordinaten fuer die Client-Tabelle (bereits vor dieser Session erledigt).

## Offene Punkte / Bei Problemen pruefen

### Hardcoded Regionen

Falls der Bot auf TM55 bei bestimmten Dialogen/Buttons nicht richtig erkennt, muessen diese Werte **lokal auf TM55** angepasst werden:

| Datei | Zeile | Aktueller Wert | Zweck |
|---|---|---|---|
| `bot_controller.py` | 339 | `region=(750,500,65,100)` | Locked-Client Dialog (blue_i_icon) |
| `config/processes/1040.json` | 162 | `[0,0,300,80]` | Clients-Button Suchbereich (oben links) |
| `config/processes/1040.json` | 170 | `[800,530,50,70]` | Save-Changes Dialog (blue_questionmark_icon) |
| `config/processes/1040.json` | 174 | `[955,615,80,30]` | "No"-Button im Save-Dialog |
| `config/processes/1120.json` | ~190 | `scroll_x:960, scroll_y:540` | Scroll-Zentrum |
| `config/processes/1120S.json` | ~146 | `scroll_x:960, scroll_y:540` | Scroll-Zentrum |

**Hinweis:** Da TM55 dieselbe Aufloesung (1920x1080) hat, sollten diese Werte passen. Falls nicht: mit `debug_click.py` die korrekte Position ermitteln und lokal aendern.

### Debugging-Tipps fuer TM55

- **Log-Datei (Exe-Modus):** `C:\Users\TaxAct\AppData\Roaming\TaxActBot\logs\bot.log` (NICHT `C:\TaxActBot\logs\bot.log`)
- **GUI-Log** zeigt nur `_send_log()` Nachrichten aus `bot_controller.py`
- **OCR-Debugging:** `debug_ocr.py` zeigt was die OCR tatsaechlich liest
- **Koordinaten-Debugging:** `debug_click.py` zum Ermitteln von Positionen
- **Temporaeres Info-Logging:** In `vision.py` Zeile ~1009 `logger.debug` auf `logger.info` aendern, um CSV-Matching-Probleme im Log zu sehen
